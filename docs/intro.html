概要¶

SQLAlchemy SQL ToolkitとObject Relational Mapperは、データベースとPythonを扱う包括的なツールセットです。それは個別にまたは一緒に組み合わせることができる機能のいくつかの異なる領域を持っています。主なコンポーネントを以下に示します。コンポーネントの依存関係はレイヤーにまとめられています。

上では、SQLAlchemyの最も重要な正面の2つの部分は** Object Relational Mapper と SQL Expression Language **です。 SQL式はORMとは独立して使用できます。 ORMを使用する場合、SQL表現言語は、オブジェクト・リレーショナル構成および問合せ内で使用されるため、パブリック・フェイシングAPIの一部として残ります。

ドキュメントの概要¶

ドキュメントは、：ref： orm_toplevel、：ref：` core_toplevel`、：ref： `dialect_toplevel`の3つのセクションに分かれています。

In：ref： `orm_toplevel`では、オブジェクトリレーショナルマッパーが導入され、完全に説明されています。新しいユーザは：ref： `ormtutorial_toplevel`で始めるべきです。あなたのために自動的に構築される上位レベルのSQLで作業したい場合や、Pythonオブジェクトの管理については、このチュートリアルに進んでください。

In：ref： `core_toplevel`では、SQLAlchemyのSQLとデータベースの統合と記述サービスの幅が文書化されており、その中核はSQL式言語です。 SQL式言語は、ORMパッケージとは独立した独自のツールキットです。このツールキットを使用して、プログラムで構築、変更、実行できる操作可能なSQL式を作成し、カーソルのような結果セットを戻すことができます。 ORMのドメイン中心の使用モードとは対照的に、表現言語はスキーマ中心の使用パラダイムを提供します。新規ユーザーは、ここでは：ref： `sqlexpression_toplevel`で始める必要があります。 SQLAlchemyのエンジンサービス、接続サービス、およびプーリングサービスについてはref： `core_toplevel`にも記述されています。

In：ref： dialect_toplevel、提供されているすべてのデータベースとDBAPIバックエンドのリファレンスドキュメントが提供されています。

コード例¶

主にORMに関する作業コードの例は、SQLAlchemyディストリビューションに含まれています。含まれているすべてのサンプルアプリケーションの説明は、：ref： `examples_toplevel`です。

また、コアSQLAlchemy構造体とwiki上のORMの両方を含む多種多様な例があります。 `Theatrum Chemicum <http://www.sqlalchemy.org/trac/wiki/UsageRecipes> `_。

インストールガイド¶

サポートされているプラ​​ットフォーム¶

SQLAlchemyは、以下のプラットフォームに対してテストされています。

  • cPython 2.7
  • cPython 3.4以上
  • `ピピ<http://pypy.org/> `_ 2.1以上

バージョン 1.2 で変更: Python 2.7がサポートされている最小のPythonバージョンになりました。

バージョン 1.3 で変更: Python 3シリーズでは、3.4がサポートされている最小のPython 3バージョンになりました。

現在サポートしていないプラットフォームには、JythonとIronPythonがあります。 Jythonは過去にサポートされており、今後のリリースでもJython自体の状態に応じてサポートされる可能性があります。

サポートされているインストール方法¶

SQLAlchemyのインストールは、 setuptools <http://pypi.python.org/pypi/setuptools/> `_、 setup.py``を直接参照するか pip <http://pypi.python.org/pypi/pip/> `_または他のsetuptools互換のアプローチ。

バージョン 1.1 で変更: setuptoolsがsetup.pyファイルで必要になりました。プレーンなdistutilsのインストールはもはやサポートされていません。

ピップ経由でインストールする¶

`` pip``が利用可能な場合、配布物はPypiからダウンロードして一段階でインストールできます:

pip install SQLAlchemy

このコマンドはPythonチーズショップからSQLAlchemyの**最新バージョン**をダウンロードします<http://pypi.python.org/pypi/SQLAlchemy> `_システムにインストールしてください。

`` 1.2.0b1``のような最新の**プレリリース版をインストールするには、 `` --pre``フラグを使用する必要があります:

pip install --pre SQLAlchemy

上記の場合、最新版がプレリリース版である場合、最新リリース版の代わりにインストールされます。

setup.pyを使ってインストールする¶

それ以外の場合は、 `` setup.py``スクリプトを使ってディストリビューションからインストールできます:

python setup.py install

C拡張機能のインストール¶

SQLAlchemyには、結果セットを処理するための追加の高速ブーストを提供するC拡張が含まれています。拡張は、cPythonの2.xxと3.xxシリーズの両方でサポートされています。

適切なプラットフォームが検出された場合、 `` setup.py``は自動的に拡張機能をビルドします。コンパイラやその他の問題が原因でC拡張のビルドが失敗した場合、セットアッププロセスは警告メッセージを出力し、完了時にC拡張を使わずにビルドを再実行し、最終ステータスを報告します。

C拡張をコンパイルせずにビルド/インストールを実行するには、 `` DISABLE_SQLALCHEMY_CEXT``環境変数を指定することができます。これのためのユースケースは特別なテスト環境のためか、まれに互換性/ビルドの問題が通常の &quot;再構築&quot;メカニズムによって克服されない場合です:

export DISABLE_SQLALCHEMY_CEXT=1; python setup.py install

バージョン 1.1 で変更: 従来の `` --without-cextensions``フラグは、setuptoolsの非推奨機能に依存しているため、インストーラから削除されました。

データベースAPIのインストール¶

SQLAlchemyは、特定のデータベース用に構築された：term： `DBAPI`実装で動作するように設計されており、最も一般的なデータベースをサポートしています。 ：doc： `/ dialects / index`の個々のデータベースセクションは、外部リンクを含む各データベースの利用可能なDBAPIを列挙します。

インストールされているSQLAlchemyのバージョンの確認¶

このドキュメントでは、SQLAlchemyバージョン1.3について説明します。すでにSQLAlchemyがインストールされているシステムで作業している場合は、Pythonプロンプトから次のようにバージョンを確認してください：

>>> import sqlalchemy
>>> sqlalchemy.__version__ 
1.3.0
