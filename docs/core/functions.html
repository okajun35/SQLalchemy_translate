SQLと汎用関数¶

SQLAlchemyがデータベース特有のレンダリング、戻り値の型、および引数の動作に関して知っているSQL関数。一般関数は：attr： `func`属性を使って、すべてのSQL関数のように呼び出されます:

select([func.count()]).select_from(sometable)

attr： `func`には知られていない任意の名前がそのまま関数名を生成することに注意してください。-SQLAlchemyに対して、組み込み関数またはユーザー定義関数で呼び出すことのできるSQL関数を制限することはできません。このセクションでは、SQLAlchemyがすでに使用されている引数と戻り値の型を知っている関数についてのみ説明します。

SQLファンクションAPI、ファクトリ、および組み込み関数が含まれます。

as_comparison(left_index, right_index)¶

この式を2つの値のブール比較として解釈します。

仮想のSQL関数&quot;is_equal（）&quot;は、等価の値と比較して、コア表現言語で次のように記述されます:

expr = func.is_equal("a", "b")

上記の&quot;is_equal（）&quot;が&quot;a &quot;と&quot;b &quot;を等価であると比較すると、：meth： `.FunctionElement.as_comparison`メソッドは:

expr = func.is_equal("a", "b").as_comparison(1, 2)

上記の場合、整数値 &quot;1&quot;は&quot;is_equal（）&quot;関数の第1引数を参照し、整数値 &quot;2 &quot;は第2引数を参照します。

これは：class： `.BinaryExpression`を生成します：これは:

BinaryExpression("a", "b", operator=op.eq)

しかし、SQLレベルでは、 &quot;is_equal（ &#39;a&#39;、 &#39;b&#39;）&quot;としてレンダリングされます。

ORMは、関連するオブジェクトまたはコレクションをロードするときに、JOIN式のON句の左辺と右辺を操作できる必要があります。このメソッドの目的は、：paramref： `.relationship.primaryjoin`パラメータとともに使用すると、この情報をORMにも渡すことができるSQL関数構造を提供することです。戻り値は、class： `.FunctionAsBinary`と呼ばれる包含オブジェクトです。

ORMの例は次のとおりです:

class Venue(Base):
    __tablename__ = 'venue'
    id = Column(Integer, primary_key=True)
    name = Column(String)

    descendants = relationship(
        "Venue",
        primaryjoin=func.instr(
            remote(foreign(name)), name + "/"
        ).as_comparison(1, 2) == 1,
        viewonly=True,
        order_by=name
    )

上の&quot;Venue &quot;クラスは、親Venueの名前が仮説的子孫値の名前の先頭に含まれているかどうかを判定することによって子孫&quot;Venue &quot;オブジェクトをロードすることができます。例えば、&quot;parent1 &quot;は&quot;parent1 / child1 &quot;ではなく、 &quot;parent2 / child1 &quot;には適用されません。

可能なユースケースには、上記の「マテリアライズドパス」の例があります。また、ジオメトリ関数などの特殊なSQL関数を使用して結合条件を作成することもできます。

パラメータ:
  • left_index¶ -- 式の左辺として機能する関数の引数の1から始まる整数のインデックス。
  • right_index¶ -- 式の右辺として機能する関数の引数の1から始まる整数のインデックス。

バージョン 1.3 で追加.

self_group(against=None)¶

Apply a 'grouping' to this ClauseElement.

This method is overridden by subclasses to return a "grouping" construct, i.e. parenthesis. In particular it's used by "binary" expressions to provide a grouping around themselves when placed into a larger expression, as well as by select() constructs when placed into the FROM clause of another select(). (Note that subqueries should be normally created using the Select.alias() method, as many platforms require nested SELECT statements to be named).

As expressions are composed together, the application of self_group() is automatic - end-user code should never need to use this method directly. Note that SQLAlchemy's clause constructs take operator precedence into account - so parenthesis might not be needed, for example, in an expression like x OR (y AND z) - AND takes precedence over OR.

The base self_group() method of ClauseElement just returns self.

class sqlalchemy.sql.functions.GenericFunction(*args, **kwargs)¶

Bases: sqlalchemy.sql.functions.Function

「汎用」関数を定義します。

ジェネリック関数は、：data： .func`属性から名前で呼ばれたときに自動的にインスタンス化される、事前設定されたclass： .Function`クラスです。 ：data： .func`から任意の名前を呼び出すと、その名前を指定するとnew：class： .Function`インスタンスが自動的に作成されるという効果があります。 a：class： `.GenericFunction`クラスを定義するための主な使用例は、特定の名前の関数に固定戻り値型を与えることができるようにすることです。カスタム引数の構文解析スキームと追加のメソッドも含めることができます。

：class： .GenericFunction`のサブクラスは自動的にクラスの名前で登録されます。例えば、ユーザ定義の関数 ` as_utc（） ``はすぐに利用できます:

from sqlalchemy.sql.functions import GenericFunction
from sqlalchemy.types import DateTime

class as_utc(GenericFunction):
    type = DateTime

print select([func.as_utc()])

ユーザ定義のジェネリック関数は、class： .GenericFunction`を定義するときに&quot; package &quot;属性を指定することで、パッケージ化することができます。多くの関数を含むサードパーティのライブラリは、他のシステムとの名前の競合を避けるためにこれを使用したいかもしれません。例えば、 ` as_utc（） ``関数が&quot;time &quot;パッケージに含まれているとします:

class as_utc(GenericFunction):
    type = DateTime
    package = "time"

上記の関数は以下から入手可能です：data： .func`パッケージ名 time`

print select([func.time.as_utc()])

最後のオプションは：data： .func`内のある名前から関数にアクセスできるようにすることですが、別の名前としてレンダリングすることです。 ` identifier``属性は：data： .func`からロードされた関数へのアクセスに使用された名前を上書きしますが、レンダリングされた名前として `name``の使用法を保持します:

class GeoBuffer(GenericFunction):
    type = Geometry
    package = "geo"
    name = "ST_Buffer"
    identifier = "buffer"

上記の関数は次のようにレンダリングされます:

>>> print func.geo.buffer()
ST_Buffer()

バージョン 0.8 で追加: ：class： `.GenericFunction`は、新しい関数の自動登録と、パッケージとカスタム命名のサポートをサポートするようになりました。

バージョン 0.8 で変更: 属性名 `` type``は、クラスレベルで関数の戻り値の型を指定するために使用されます。以前は、 `` __return_type__``という名前が使われていました。この名前は下位互換性のために認識されています。
