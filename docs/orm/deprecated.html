非推奨のORMイベントインターフェイス¶

このセクションでは、SQLAlchemy 0.1にはじめて登場したクラスベースのORMイベントインターフェイスについて説明します。これは、SQLAlchemy 0.5までさらに多くの種類のイベントが進行しました。非ORMアナログは：ref： `dep_interfaces_core_toplevel`に記述されています。

バージョン 0.7 で撤廃: SQLAlchemy 0.7以降、：ref： `event_toplevel`に記述された新しいイベントシステムは、拡張/プロキシ/リスナーシステムを置き換え、サブクラス化を必要とせずにすべてのイベントに対して一貫したインターフェースを提供します。

マッパーイベント¶

class sqlalchemy.orm.interfaces.MapperExtension¶

class： `.Mapper`イベントフックの基本実装です。

注釈

：class： .MapperExtension`は推奨されていません。 func： `.event.listen`と：class： .MapperEvents`を参照してください。

新しい拡張クラスsubclass：class： .MapperExtension`そして extension` mapper（）引数を使用して指定されます。この引数はclass：` .MapperExtension`またはそのようなリストです：

from sqlalchemy.orm.interfaces import MapperExtension

class MyExtension(MapperExtension):
    def before_insert(self, mapper, connection, instance):
        print "instance %s before insert !" % instance

m = mapper(User, users_table, extension=MyExtension())

単一のマッパーは、 `` MapperExtension``オブジェクトの連鎖を維持することができます。特定のマッピングイベントが発生すると、各「MapperExtension」に対応するメソッドが順次呼び出され、各メソッドはチェーンをさらに進めるのを止めることができます:

m = mapper(User, users_table, extension=[ext1, ext2, ext3])

それぞれの `` MapperExtension``メソッドは、デフォルトでシンボルEXT_CONTINUEを返します。このシンボルは一般に、&quot;を処理するために次の` `MapperExtension``に移動することを意味します。変換された行や新しいオブジェクトインスタンスのようなオブジェクトを返すメソッドの場合、EXT_CONTINUEはメソッドの結果を無視することを意味します。場合によっては、結果リストに新しいインスタンスを追加するなど、デフォルトのマッパーアクティビティーを実行する必要があります。

シンボルEXT_STOPは、 `` MapperExtension``オブジェクトのチェーン内で、このシンボルが返されたときにチェーンが停止する意味を持ちます。 EXT_CONTINUEと同様に、デフォルトのマッパーアクティビティーが実行されない場合もあります。

after_delete(mapper, connection, instance)¶

インスタンスが削除された後にオブジェクトインスタンスを受け取ります。

戻り値は `` MapperExtension``チェーン内でのみ重要です。親マッパーの動作はこのメソッドによって変更されません。

after_insert(mapper, connection, instance)¶

インスタンスが挿入された後にオブジェクトインスタンスを受け取ります。

戻り値は `` MapperExtension``チェーン内でのみ重要です。親マッパーの動作はこのメソッドによって変更されません。

after_update(mapper, connection, instance)¶

インスタンスが更新された後にオブジェクトインスタンスを受け取ります。

戻り値は `` MapperExtension``チェーン内でのみ重要です。親マッパーの動作はこのメソッドによって変更されません。

before_delete(mapper, connection, instance)¶

インスタンスが削除される前にオブジェクトインスタンスを受け取ります。

ここでは、全体的なフラッシュプランの変更はできません。 `` Session``の操作には望みの効果はありません。拡張子の中で `` Session``を操作するには、 `` SessionExtension``を使います。

戻り値は `` MapperExtension``チェーン内でのみ重要です。親マッパーの動作はこのメソッドによって変更されません。

before_insert(mapper, connection, instance)¶

インスタンスがそのテーブルに挿入される前にオブジェクトインスタンスを受け取ります。

これはプライマリキーの値を設定するのに適しており、それ以外の場合は処理されません。

このメソッド内で列ベースの属性を変更すると、新しい値が挿入されます。しかし、全体的なフラッシュプランの変更はできません。また、 `` Session``の操作は望みの効果を持たないでしょう。拡張子の中で `` Session``を操作するには、 `` SessionExtension``を使います。

戻り値は `` MapperExtension``チェーン内でのみ重要です。親マッパーの動作はこのメソッドによって変更されません。

before_update(mapper, connection, instance)¶

インスタンスが更新される前にオブジェクトインスタンスを受け取ります。

このメソッドは、列ベースの属性に正味の変更を加えていない場合でも、&quot;ダーティ&quot;とマークされているすべてのインスタンスに対して呼び出されることに注意してください。列ベースの属性のいずれかが呼び出されたときに、またはそのコレクションのいずれかが変更されたときに、オブジェクトはダーティとしてマークされます。更新時に列ベースの属性に正味の変更がない場合、UPDATEステートメントは発行されません。つまり、before_updateに送信されるインスタンスは、UPDATE文が発行される保証ではありません（ただし、ここでの結果に影響を与える可能性があります）。

オブジェクトの列ベースの属性に正味の変更があるかどうかを検出し、UPDATE文を生成するかどうかを検出するには、 `` object_session（instance）.is_modified（instance、include_collections = False） ``を使用します。

このメソッド内で列ベースの属性を変更すると、新しい値が更新されます。しかし、全体的なフラッシュプランの変更はできません。また、 `` Session``の操作は望みの効果を持たないでしょう。拡張子の中で `` Session``を操作するには、 `` SessionExtension``を使います。

戻り値は `` MapperExtension``チェーン内でのみ重要です。親マッパーの動作はこのメソッドによって変更されません。

init_failed(mapper, class_, oldinit, instance, args, kwargs)¶

コンストラクタが呼び出されたときにインスタンスを受け取り、例外を送出します。

このメソッドは、オブジェクトのユーザランド構築時にのみ呼び出されます。オブジェクトがデータベースからロードされたときには呼び出されません。

戻り値は `` MapperExtension``チェーン内でのみ重要です。親マッパーの動作はこのメソッドによって変更されません。

init_instance(mapper, class_, oldinit, instance, args, kwargs)¶

コンストラクタが呼び出されたときにインスタンスを受け取ります。

このメソッドは、オブジェクトのユーザランド構築時にのみ呼び出されます。オブジェクトがデータベースからロードされたときには呼び出されません。

戻り値は `` MapperExtension``チェーン内でのみ重要です。親マッパーの動作はこのメソッドによって変更されません。

instrument_class(mapper, class_)¶

マッパーが最初に構築され、マッピングされたクラスに計測を適用したときにクラスを受け取ります。

戻り値は `` MapperExtension``チェーン内でのみ重要です。親マッパーの動作はこのメソッドによって変更されません。

reconstruct_instance(mapper, instance)¶

オブジェクトインスタンスが `` __new__``を介して生成され、初期属性の集合が発生した後にそれを受け取ります。

これは、通常、受信結果の行に基づいてインスタンスが作成され、そのインスタンスの存続時間に対して一度だけ呼び出される場合に発生します。

結果行のロード中、このメソッドはこのインスタンスに対して最初に受信された行で呼び出されることに注意してください。結果行にあるものに応じて、属性やコレクションの一部がロードされたり、初期化されたりされたりすることがあります。

戻り値は `` MapperExtension``チェーン内でのみ重要です。親マッパーの動作はこのメソッドによって変更されません。

セッションイベント¶

class sqlalchemy.orm.interfaces.SessionExtension¶

class： `.Session`イベントフックの基本実装です。

注釈

：class： .SessionExtension`は非推奨です。 func： `.event.listen`と：class： .SessionEvents`を参照してください。

サブクラスは `` extension``キーワード引数を使って：class： .Session`（または：class： .sessionmaker`）にインストールすることができます:

from sqlalchemy.orm.interfaces import SessionExtension

class MySessionExtension(SessionExtension):
    def before_commit(self, session):
        print "before commit!"

Session = sessionmaker(extension=MySessionExtension())

同じ：class： `.SessionExtension`インスタンスは任意の数のセッションで使用できます。

after_attach(session, instance)¶

インスタンスがセッションにアタッチされた後に実行します。

これは、追加、削除、またはマージの後に呼び出されます。

after_begin(session, transaction, connection)¶

接続時にトランザクションが開始された後に実行する

`transaction`はSessionTransactionです。このメソッドは、接続時にエンジンレベルのトランザクションが開始された後に呼び出されます。

after_bulk_delete(session, query, query_context, result)¶

セッションに対する一括削除操作の後に実行します。

これはsession.query（...）の後に呼び出されます。delete（）

`query`は、この削除オペレーションが呼び出されたクエリオブジェクトです。 `query_context`はクエリコンテキストオブジェクトでした。 `result`は、バルク操作から返された結果オブジェクトです。

after_bulk_update(session, query, query_context, result)¶

セッションに対する一括更新操作の後に実行します。

これはsession.query（...）の後に呼び出されます。update（）

`query`は、この更新オペレーションが呼び出されたクエリオブジェクトです。 `query_context`はクエリコンテキストオブジェクトでした。 `result`は、バルク操作から返された結果オブジェクトです。

after_commit(session)¶

コミットが発生した後に実行します。

これは、より長い実行中のトランザクションが進行中であれば、1フラッシュではないことに注意してください。

after_flush(session, flush_context)¶

flushが完了した後、commitが呼び出される前に実行します。

セッションの状態はまだpre-flushであることに注意してください。つまり、 &#39;new&#39;、 &#39;dirty&#39;、 &#39;deleted&#39;リストはプリフラッシュ状態とインスタンス属性の履歴設定を表示します。

after_flush_postexec(session, flush_context)¶

フラッシュが完了した後、および実行後状態が発生した後に実行します。

これは、「新規」、「ダーティ」、および「削除済み」リストが最終状態にあるときです。フラッシュが自身のトランザクションを開始したかどうか、またはより大きなトランザクションに参加したかどうかによって、実際のcommit（）が発生しているかどうかは不明です。

after_rollback(session)¶

ロールバックが発生した後に実行します。

これは、より長い実行中のトランザクションが進行中であれば、1フラッシュではないことに注意してください。

before_commit(session)¶

コミットが呼び出される直前に実行します。

これは、より長い実行中のトランザクションが進行中であれば、1フラッシュではないことに注意してください。

before_flush(session, flush_context, instances)¶

フラッシュ処理が開始される前に実行します。

instances`は、 flush（） `メソッドに渡されたオブジェクトのオプションリストです。

属性イベント¶

class sqlalchemy.orm.interfaces.AttributeExtension¶

：class： `.AttributeImpl`イベントフック、ユーザコードの属性の突然変異を引き起こすイベント。

注釈

：class： .AttributeExtension`は推奨されていません。 func： `.event.listen`と：class： .AttributeEvents`を参照してください。

：class： .AttributeExtension`は個々のマップされた属性のset、remove、appendイベントをリスンするために使われます。 func： `.column_property、：func：` .relationship`などで利用できる `extension`引数を使用して、個々のマップされた属性に設定されます。

from sqlalchemy.orm.interfaces import AttributeExtension
from sqlalchemy.orm import mapper, relationship, column_property

class MyAttrExt(AttributeExtension):
    def append(self, state, value, initiator):
        print "append event !"
        return value

    def set(self, state, value, oldvalue, initiator):
        print "set event !"
        return value

mapper(SomeClass, sometable, properties={
    'foo':column_property(sometable.c.foo, extension=MyAttrExt()),
    'bar':relationship(Bar, extension=MyAttrExt())
})

class： .AttributeExtension`メソッド：meth：〜.AttributeExtension.append`と：meth： 〜.AttributeExtension.set`は、 `value``パラメータを返す必要があることに注意してください。戻り値は実効値として使用され、最終的に永続化されるものを拡張が変更できるようにします。

AttributeExtensionは、マップされたクラスに関連付けられた記述子内でアセンブルされます。

active_history = True¶

は、たとえそれが遅延呼び出し可能呼び出しを意味するとしても、set（）メソッドが &#39;old&#39;値を受け取ることを希望していることを示します。

`` active_history``は：func： .column_property`と：func： .relationship`を介して直接設定することもできます。

append(state, value, initiator)¶

コレクションの追加イベントを受け取ります。

返された値は、追加される実際の値として使用されます。

remove(state, value, initiator)¶

removeイベントを受け取ります。

戻り値は定義されていません。

set(state, value, oldvalue, initiator)¶

設定されたイベントを受信します。

返される値は、実際に設定される値として使用されます。
